Movie API -Documentation

To build the server­side component of a “movies” web application, this API provides a database that stores data about different movies,
providing users access to information about their different directors and genres.

The table below describes the endpoints of this API and which Information will be send back and forward:

Description URL HTTP method Request body data format Response body data format
Get a list of all movies /movies GET none JSON object holding data of all movies from list.
Get information about a certain movie /movies/[title] GET none JSON object holding data of requested film-title. Example:
{
 title: "Casablanca",
 description: "World War II-era movie, etc",
 genre: {
  name:"Drama",
  description: "Movie with serious undertones, etc.“
  },
director: {
  name: "Michael Curtiz",
  bio: "Hungarian-born American Director, etc.",
  birth: "1886",
  death: "1962“
  },
imagePath: "casablanca.png",
featured: false
}
Get information about a certain genre /movies/genre/[name] GET none JSON object holding data of requested film-genre. Example:
{
name:"Drama",
description: "Movie with serious undertones, etc.“
}
Get information about a certain director /movies/director/[name] GET none JSON object holding data of requested director. Example:
{
name: "Michael Curtiz",
bio: "Hungarian-born American Director, etc.",
birth: "1886",
death: "1962“
}
Add a new user to the registry /users POST JSON object holding data of the user to add:
{
 username: “myName”,
 password: ”myPassword”,
 email:”mail@gmail.com”,
 date of birth: “29-09-1985”
} 		JSON object holding data of the user added to registry
including newly allocated user-id. Example:
{
id: 5,
username:”myName”,
password:”myPassword”,
email:”mail@gmail.com”,
date of birth:”29-09-1985”
}
Update ones user information /users/[username]/profile PUT JSON object holding data of the user to add:
{
 username: “newName”,
 password: ”newPassword”,
 email:”new@gmail.com”,
 date of birth: “29-09-90”
} 		A text message indicating user information that was updated to profile.
Add new movie to the users favorite-list /users/[username]/favorites POST none Text message indicating that movie has been added to favorite-list.
Delete a movie from the users favorite-list /users/[username]/favorites/[title] DELETE none Text message that movie has successfully removed from favorite-list.
Remove a user from the the registry /users/[username] DELETE none Text message whether user has successfully been removed.
